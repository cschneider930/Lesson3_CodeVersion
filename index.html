Hypertext Transfer Protocol What is HTTP?

Hypertext transfer protocol (HTTP) at its core can be seen as a data delivery service. It is the communication between servers and computers and transfers text, commands, and any other kind of information given that both sides know how to send and receive it. It is called a stateless protocol because it does not retain information. Each query made by the user is sent out with no reference or memory of past requests. Though this isn't the flashiest or most visible part of web access it is by far one of the most vital.

Hypertext Markup Language What is HTML?

Hypertext markup language is the code behind webpages. Though design elements and other features can be enhanced with other tools and languages such as CSS and JavaScript, HTML is the core of any page and a perfectly functional page can be created with just HTML. Tags define the structure of a page and commonly split it into two main sections, the head and the body. This language is quick to learn and easy enough to use while hand coding, however users have the option of using a WYSIWYG (What you see is what you get) editor.

Development Cycle Ask questions first

Many believe that creating a website is as simple as having an idea, buying a domain, and writing some code. However, by taking some time to consider the phases of the web development life cycle developers create more efficient websites that are better suited for the needs of their audience and have greater longevity. The phases are planning, analysis, design and development, testing, and implementation and maintenance. This process emphasizes communication throughout the design and development of the site so the developer and client are on the same page as to the needs and expectations.
